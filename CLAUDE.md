## Architecture

**Stack:** Next.js 16 (App Router) + Convex (serverless backend) + Clerk (auth) + shadcn/ui + Tailwind CSS v4

### Key Layers

- **`app/`** — Next.js App Router pages. `layout.tsx` wraps the app with `ClerkProvider` (auth) and `ThemeProvider` (dark mode). React Server Components by default.
- **`convex/`** — Serverless backend with real-time data syncing. `auth.config.ts` integrates Clerk JWT tokens. `_generated/` is auto-generated by the Convex CLI — never edit manually.
- **`components/ui/`** — 57 shadcn/ui components built on Radix UI primitives. Import as `import { Button } from "@/components/ui/button"`.
- **`proxy.ts`** — Clerk authentication middleware (Next.js middleware).

### Styling

- Tailwind CSS v4 (uses `@import` directives, not `@tailwind` directives)
- CSS variables in `app/globals.css` using oklch color format for theming
- Dark mode via class strategy with `next-themes`
- Use `cn()` from `@/lib/utils` for conditional class merging (clsx + tailwind-merge)
- `cva()` for component variants

### Path Aliases

`@/*` maps to the project root.

### Environment Variables

Required in `.env.local`:
- `CONVEX_DEPLOYMENT` + `NEXT_PUBLIC_CONVEX_URL` + `NEXT_PUBLIC_CONVEX_SITE_URL` — Convex backend
- `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` + `CLERK_SECRET_KEY` + `CLERK_JWT_ISSUER_DOMAIN` — Clerk auth
